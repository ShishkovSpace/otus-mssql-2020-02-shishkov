Описание проекта:
Данный проект представляет собой БД, в которой хранятся результаты нагрузочного тестирования web приложения для различных проектов.
Основная идея и решаемая задача - хранить данные по нагрузочному тестированию для различных версий приложения, выполненных в различных инфраструктурных условиях. 
Загрузка данных по проведенным тестам предполагается из файлов .csv.
Тестирования возможно для различных проектов и на различных версиях развернутого приложения. Также в рамках одного проекта проведение тестов возможно на различных сборках одной и той же версии.
Информация по проектам, версиям и сборкам приложения содержится в таблице LT.ProjectInfo.
Таблица LT.EnvInfo содержит информацию об окружении, на котором проводится тестирование. В данную информацию могут входит количество инстансов приложения, указатель на работу в облаке и на тестирование на внутреннем окружении.
Информация по тестовым сценариям находится в табличке LT.ScenarioInfo. На текущем этапе проекта предполагается, что список критериев, по которым проводится тестирование и анализ - статичен, поэтому в таблицах LT.ScoresLT_XX предполагается хранение результатов по каждому из критериев соответственно для каждого из выполненных сценариев. То есть в процессе тестирования web приложения выполняются запросы к сервисам back-end, результаты выполнения которых предполагается хранить в вышеуказанных таблицах с учетом группировки для каждого критерия, т.к. некоторые из критериев могут состоять и оцениваться как группа выполненных запросов.
В дальнейшем, по полученным результатам для каждого из критерия предполаагется выполнять анализ с целью получения статистических данных, таких как количество выполнений, количество ошибок, среднее, минимальное и максимальное времена выполнения (возможно 90-й перцентиль и т.д.) и дальнейшего сравнивания полученных показателей с целевыми. Агрегированную информацию по проведенным тестам предполагается хранить в таблице LT.Scores_AGR. Информацию о целевых показателях для каждого критерия в рамках конкретного проекта хранится в таблице LT.TargetInfo.
Предполагается, что независимо от результатов сравнения с целевыми показателями, необходимо хранить информацию по каждому HTTP запросу выполенному в процессе тестирования, чтобы иметь возможность анализировать динамику выполнения теста, и конкретные причины и условия возможных неполадок. Для доступа к данной информации и возможности реализации данного функционала создана таблица LT.ArchiveLT.

Ориентировочный объем при разовой загрузке данных для таблиц, хранящих результаты по критериям (LT.ScoresLT_XX) - 9-10 тысяч записей, для архивной информации по всем запросам, выполненным в тесте (LT.ArchiveLT) - 150-200 тысяч записей. 
Загрузку данных предполагается выполнять в рамках одной транзакции, выполняемой хранимой процедурой.
